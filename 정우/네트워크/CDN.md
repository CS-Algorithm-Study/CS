## 🤔 CDN 이란?
</br>

**CDN(Content Delivery Network)이란 컨텐츠 전송 네트워크로, 컨텐츠를 효율적으로 전달하기 위해 여러 노드를 가진 네트워크(CDN 서버)에 컨텐츠를 저장하여 제공하는 시스템이다**

</br>

- 본 서버의 컨텐츠들을 CDN 서버에 캐싱해두고 사용자가 요청할 때, CDN 서버에서 정적, 동적 컨텐츠를 제공하여 응답속도를 향상시킴
  
</br>

### 정적 컨텐츠와 동적 컨텐츠

- **정적 컨텐츠**
  - HTML, CSS, JS, Image
  - 정적 컨텐츠는 변경이 자주 일어나지 않기 때문에 미리 CDN 서버에 보내 놓는다 **(정적 캐싱)**

</br>

- **동적 컨텐츠**
  - 사용자에 요청에 따라 매번 바뀌는 컨텐츠
  - 사용자의 요청한 데이터가 CDN에 있는지 확인하고 있다면 보내고(Cache hit), 없다면 (Cache miss) 본 서버에서 받아와 전달한다 **(동적 캐싱)**
  - 동적 컨텐츠의 특성상 캐싱에 어려움이 있다. 하지만 바이트 단위로 분석하여 바뀐 부분만 받아오거나 데이터 압축, 서버와의 통신과정(handshake)를 간소화 하여 보완하기도 한다.
    
</br>

## 🤔 CDN 처리 과정

</br>

![image](https://github.com/CS-Algorithm-Study/CS/assets/81271328/c0b34fa6-aaf0-47c9-be39-3a8cbcb0391a)

- 사용자가 웹 브라우저를 통해 url 주소를 입력
- DNS 서버는 가장 가깝거나 트래픽이 적은 CDN 서버의 IP 주소를 반환
- CDN 서버에 클라이언트 HTTP 를 요청
- CDN 에 컨텐츠가 존재하지 않는 경우 본 서버에 요청
- 본 서버에서 CDN 서버에 컨텐츠를 전송하고, CDN은 이를 캐싱한다 (다음 방문자는 캐싱된 컨텐츠를 제공받을 수 있다)

</br>

## 🤔 CDN 장점

</br>

- **페이지 로딩 시간 단축**
  - CDN 서버에서 제공하는 컨텐츠를 이용하여 페이지 로딩 시간을 단축시킬 수 있다
  - 또한 본 서버와 CDN 서버 간의 통신은 신뢰할 수 있기에 연결 과정이 단순화 되어있고, 따라서 컨텐츠를 주고 받는 속도가 빠르다

- **트래픽 분산**
  - CDN을 통해 여러 컨텐츠들이 제공되므로 서버의 부하를 줄일 수 있다   
  - 본 서버로 요청되는 트래픽을 CDN에서 분산시킨다

- **보안 강화**
  -  DDOS 공격으로 인해 발생되는 많은 트래픽을 CDN 서버에 의해 분산되기 때문에 한 서버에 트래픽이 몰리는 것을 방지할 수 있다
  -  CDN 서버를 제공하는 업체 자체에서 보안 기능을 제공
